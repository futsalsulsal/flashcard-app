# フラッシュカード学習アプリ 改善要件定義書

## 前提条件・制約

- **HTML1ファイル + CSVカートリッジの構造を維持する**
- 外部サーバー・アカウント・外部ライブラリは使わない
- 学習状態はすべてlocalStorageに保存する（セーブデータの位置づけ）
- CSVはあくまで初期データの入力手段であり、学習状態を書き戻さない

---

## 改善1: 間隔反復（Spaced Repetition）

### 概要

忘却曲線に基づき、各カードに「次回復習日」を設定する。
ユーザーは「○日前に覚えた単語」のようにフィルタリングして学習できる。

### 忘却曲線の基準タイミング

初回学習後、以下のタイミングで復習を促す:

| 復習回数 | 「覚えた」を押し続けた場合の間隔 |
|---------|-------------------------------|
| 1回目 | 1日後 |
| 2回目 | 3日後 |
| 3回目 | 7日後 |
| 4回目 | 14日後 |
| 5回目 | 30日後 |
| 6回目以降 | 60日後（上限） |

※ 3段階評価の結果によって間隔が変動する（後述）。

### データ構造（localStorage）

キー: `fc_srs_{dataKey}` に、カードごとのSRS状態をJSON配列で保存する。

```json
{
  "cardId": 1,
  "status": "learning",
  "interval": 3,
  "ease": 2.5,
  "dueDate": "2026-03-30",
  "lastReview": "2026-03-27",
  "reps": 2,
  "lapses": 0
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| cardId | number | CSVの行番号に対応するID |
| status | string | `"new"` / `"learning"` / `"mastered"` |
| interval | number | 現在の復習間隔（日数） |
| ease | number | 容易さ係数（初期値2.5、最小1.3） |
| dueDate | string | 次回復習日（YYYY-MM-DD） |
| lastReview | string | 最終復習日（YYYY-MM-DD） |
| reps | number | 連続正答回数 |
| lapses | number | 「もう一度」を押した累計回数 |

### ステータス遷移

```
new（未学習）
 │
 ├─ 初めて答えを見た
 ↓
learning（学習中）
 │
 ├─ interval が 30日以上に達した
 ↓
mastered（定着済み）
 │
 ├─ 「もう一度」を押した
 ↓
learning に戻る
```

### UI: フィルタリング

現状のカテゴリフィルタに加えて、復習状態でフィルタできるようにする。

表示するフィルタボタン:

| フィルタ | 条件 |
|---------|------|
| 今日の復習 | `dueDate <= 今日` のカード（デフォルト表示） |
| 未学習 | `status === "new"` |
| 学習中 | `status === "learning"` |
| 定着済み | `status === "mastered"` |
| すべて | 全カード表示 |

### UI: 復習予定の可視化

プログレスバーの下に、復習スケジュールのサマリーを表示する。

```
今日の復習: 12語 ｜ 明日: 8語 ｜ 未学習: 45語
```

## 改善2: 3段階評価

### 概要

答えを確認した後に、3段階の自己評価ボタンを表示する。
**答えを見る前には評価ボタンを表示しない**（アクティブリコールの強制）。

### 評価ボタン

| ボタン | 色 | 間隔への影響 | ease への影響 |
|-------|-----|------------|--------------|
| もう一度 | 赤 `#ef4444` | 間隔を1日にリセット、同セッション内で再出題 | ease -= 0.2（最小1.3） |
| あやしい | 黄 `#f59e0b` | `interval × 1.2` | ease 変動なし |
| 覚えた | 緑 `#10b981` | `interval × ease` | ease += 0.1（最大3.0） |

### 操作フロー

```
1. カード表示（答えは隠れている）
     │
2. ユーザーが頭の中で答えを想起
     │
3. 「クリックで表示」を押す / カードをタップ
     │
4. 答えが表示される
     │
5. ★ 3つの評価ボタンが出現 ★
   [もう一度]  [あやしい]  [覚えた]
     │
6. いずれかを押す → SRSデータ更新 → 次のカードへ
```

### 「もう一度」の再出題ロジック

「もう一度」を押したカードは、同セッション内のキューの末尾に再追加する。
セッション終了後は `dueDate = 翌日` として保存する。

### 既存の「覚えた」ボタンとの関係

- 現在の「覚えた」→アーカイブの仕組みは**廃止**する
- 代わりに、`status === "mastered"` のカードは自動的に復習頻度が下がる
- アーカイブタブは「定着済み」タブにリネームし、mastered のカードを一覧表示する
- 定着済みカードは復習期日が来たら自動的に「今日の復習」に再出現する

## 改善3: シャッフル

### 概要

カードの表示順をシャッフルできる。ワンタップで元の順序に戻せる。

### デフォルト順序

CSVの行番号（`card.id`）をデフォルトの並び順とする。
これは既存の実装でCSV読み込み時に `id: r`（行番号）を振っているため、追加のデータ構造は不要。

### UI

ヘッダーまたは一括操作エリアにトグルボタンを配置する。

```
[🔀 シャッフル]  ← 押すとシャッフル状態になる
[↕️ 元の順序]    ← 押すとCSVの並び順に戻る
```

ボタンは状態に応じてトグルする（同じボタンが切り替わる）。

### シャッフルロジック

- Fisher-Yatesアルゴリズムで配列をシャッフルする
- シャッフル結果はメモリ上のみで保持し、localStorageには保存しない
  - 理由: セッションごとに異なる順序で出題される方が学習効果が高い
- カテゴリフィルタ適用後の表示対象カードに対してシャッフルを適用する
- フィルタ切り替え時は再シャッフルする

### シャッフルと間隔反復の関係

- 「今日の復習」表示時は、デフォルトでシャッフル状態にする
  - 復習モードでは順序のバイアスを排除するのが望ましいため
- 「すべて」「未学習」等のフィルタ時は、デフォルトは元の順序

## タブ構成の変更

### 現状

```
[ 用語→意味 | 意味→用語 | アーカイブ ]
```

### 変更後

```
[ 用語→意味 | 意味→用語 | 定着済み ]
```

- 「アーカイブ」を「定着済み」にリネーム
- 「定着済み」タブには `status === "mastered"` のカードを表示
- 定着済みタブ内にも「戻す」ボタンは残す（`status` を `"learning"` にリセット）

## データ保存の全体設計

```
localStorage
├── fc_dataset_list          # データセット名の一覧 (既存)
├── fc_data_{dataKey}        # CSV原文 (既存)
├── fc_active_dataset        # 選択中のデータセット名 (既存)
├── fc_srs_{dataKey}         # ★新規: カードごとのSRS状態
└── fc_stats_{dataKey}       # ★新規: 学習統計ログ
```

### fc_srs_{dataKey} の形式

```json
{
  "1": { "status": "learning", "interval": 3, "ease": 2.5, "dueDate": "2026-03-30", "lastReview": "2026-03-27", "reps": 2, "lapses": 0 },
  "2": { "status": "new", "interval": 0, "ease": 2.5, "dueDate": null, "lastReview": null, "reps": 0, "lapses": 0 },
  ...
}
```

cardId をキーとしたオブジェクトで保持する（配列より検索が速い）。

### fc_stats_{dataKey} の形式

```json
[
  { "date": "2026-03-27", "reviewed": 25, "again": 3, "hard": 7, "good": 15, "newLearned": 10 },
  { "date": "2026-03-28", "reviewed": 18, "again": 1, "hard": 5, "good": 12, "newLearned": 5 }
]
```

## 既存機能への影響

| 既存機能 | 変更内容 |
|---------|---------|
| CSVアップロード | 変更なし |
| CSV切り替え | 変更なし |
| カテゴリフィルタ | SRSフィルタと並列で動作（AND条件） |
| 「クリックで表示」 | 表示後に3段階評価ボタンが追加表示される |
| 「覚えた」ボタン | 廃止 → 3段階評価に置き換え |
| アーカイブ/戻す | 「定着済み」タブにリネーム、手動戻しは維持 |
| プログレスバー | mastered率に変更 |
| すべて開く/閉じる | 維持 |
| PWA/Service Worker | 変更なし |
| Shift_JIS表記 | 今回のスコープ外（別途対応） |

---

## スコープ外（将来検討）

以下は今回の実装に含めない:

- 学習統計ダッシュボード（統計データは保存するが、表示UIは後回し）
- マルチデバイス同期（JSON書き出し/読み込み）
- Shift_JIS対応
- テストモード（入力して正誤判定）
- 音声・画像対応
